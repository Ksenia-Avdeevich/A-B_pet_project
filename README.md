# Анализ результатов A/B-теста: Влияние информации о кэшбеках на конверсию дебетовой карты

**Автор:** Ксения Авдеевич  
**Тип проекта:** Пет-проект (A/B тестирование)  
**Инструменты:** Python (pandas, numpy, scipy, matplotlib, seaborn)
---

## Описание проекта

В рамках проекта анализируются результаты A/B-теста, проведенного для дебетовой карты Т-Банка. На лендинг была добавлена информация о дополнительных кэшбеках от партнеров, и основная цель — проверить, привело ли это нововведение к статистически значимому увеличению конверсии (оформление карты).
Результаты анализа необходимы для принятия решения о раскатке нового функционала на всех пользователей.

## Данные

*   **Источник:** Логи переходов на лендинг и оформления карт за 30 дней.
*   **Объем:** 94,778 записей, 4 колонки.
*   **Структура:** Дата визита, уникальный ID пользователя, группа (контроль/тест), факт конверсии.
  ---

## Метрики и гипотезы

### Основная метрика
*   **Конверсия (Conversion Rate):** Доля пользователей, оформивших дебетовую карту после посещения лендинга.

### Статистические гипотезы
*   **H₀ (Нулевая гипотеза):** Конверсия в тестовой группе (с кэшбеками) **<=** конверсии в контрольной группе (без кэшбеков). Нововведение не работает или ухудшает показатель.
*   **H₁ (Альтернативная гипотеза):** Конверсия в тестовой группе **>** конверсии в контрольной группе. Добавление информации о кэшбеках **увеличивает конверсию**.

---

## Результаты и выводы

### 1. Валидация теста
*   **Разбивка на группы:** Корректна. Размер групп сбалансирован (Контроль: 47,330, Тест: 47,448), что подтверждает корректность проведения эксперимента.
*   **Динамика по дням:** Тестовая группа стабильно показывает более высокую конверсию на протяжении всего периода. Аномалии (всплески/провалы) синхронны для обеих групп, что указывает на внешние факторы, а не на баги теста.

### 2. Ключевые показатели
*   **Контрольная группа (Control):** Конверсия составила **10.4%** (4,922 оформления).
*   **Тестовая группа (Test):** Конверсия составила **11.96%** (5,674 оформления).
*   **Абсолютный прирост:** **+1.56 п.п.**
*   **Относительный прирост (Uplift):** **+14.99%**

### 3. Статистическая значимость (Гипотезы) — *H₀ ОТВЕРГНУТА*
Результаты были проверены двумя методами для повышения надежности выводов:

| Тест | p-value | Уровень значимости | Интерпретация |
| :--- | :--- | :--- | :--- |
| **Биномиальный тест** (сравнение долей) | < 0.001 | α = 0.001 (99.9%) | **Крайне значимо** |
| **t-тест** (сравнение средних дневных конверсий) | < 0.01 | α = 0.01 (99%) | **Высоко значимо** |

*   **Вердикт:** Разница в конверсиях между группами **статистически значима** и **не является случайной**.

### 4. Что делать дальше
1.  **Ближайшие действия:**
    *   **Раскатить блок с информацией о кэшбеках на 100% пользователей.**
    *   Ожидаемый эффект: дополнительные **~23 оформления карты в день** при текущем трафике.
2.  **Следующие шаги:**
    *   **Мониторинг:** Убедиться в устойчивости эффекта в течение длительного периода (30+ дней).
    *   **Сегментный анализ:** Протестировать эффективность блока на разных сегментах (мобайл/десктоп, новые/старые пользователи).
    *   **A/B тест:** Оптимизировать дизайн и контент блока (разные наборы партнеров, визуальное оформление).
    *   **Анализ LTV:** Оценить качество привлеченных клиентов (активация карты, средний чек, отток), чтобы убедиться, что рост конверсии не происходит за счет "дешевой" аудитории.

## Визуализации

В проекте использованы ключевые типы графиков для A/B-анализа:

*   **Динамика по дням (Line plot):** Для отслеживания стабильности эффекта и выявления аномалий.
*   **Гистограмма распределения (Histogram):** Для проверки гипотезы о нормальности распределения дневных конверсий (ЦПТ).
*   **Доверительные интервалы (Error bar):** Для оценки точности точечных оценок конверсии в каждой группе.
*   **График значимости (Bar chart):** Для наглядного сравнения p-value различных статистических тестов.
